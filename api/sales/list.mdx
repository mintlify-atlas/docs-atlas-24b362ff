---
title: "List Sales"
api: "GET /sales"
description: "Retrieve all sales with detailed product and client information"
---

This endpoint returns a comprehensive list of all sale details, including information about the products, clients, quantities, prices, and totals. Each row represents a product within a sale.

## Response Format

The response includes detailed information from joined tables (sales, sale_details, clients, and products).

## Response Fields

<ResponseField name="error" type="boolean">
  Indicates if an error occurred
</ResponseField>

<ResponseField name="data" type="array">
  Array of sale detail objects
  
  <Expandable title="data properties">
    <ResponseField name="idSales" type="string">
      UUID of the sale order
    </ResponseField>
    
    <ResponseField name="idSalesDetails" type="string">
      UUID of the sale detail record (represents one product in the sale)
    </ResponseField>
    
    <ResponseField name="clientsId" type="string">
      UUID of the client who made the purchase
    </ResponseField>
    
    <ResponseField name="clientsName" type="string">
      Name of the client
    </ResponseField>
    
    <ResponseField name="idProduct" type="string">
      UUID of the product
    </ResponseField>
    
    <ResponseField name="nameProduct" type="string">
      Name of the product
    </ResponseField>
    
    <ResponseField name="quantity" type="integer">
      Quantity purchased
    </ResponseField>
    
    <ResponseField name="price" type="string">
      Unit price of the product at the time of sale
    </ResponseField>
    
    <ResponseField name="total" type="string">
      Total amount for this product (price × quantity)
    </ResponseField>
  </Expandable>
</ResponseField>

## Understanding the Response

- **Multiple rows per sale**: If a sale includes multiple products, each product appears as a separate item in the array with the same `idSales` but different `idSalesDetails`.
- **Price snapshot**: The `price` field reflects the product's price at the time of sale, which may differ from the current product price.
- **Calculated totals**: The `total` field is automatically calculated as `price × quantity`.
