---
title: "Get Sale"
api: "GET /sales/:id"
description: "Retrieve a specific sale with all product details"
---

This endpoint returns detailed information about a specific sale order, including all products purchased, client information, and pricing details.

## Path Parameters

<ParamField path="id" type="string" required>
  UUID of the sale to retrieve
</ParamField>

## Response Format

The response contains all products associated with the sale, with complete client and product information from joined tables.

## Response Fields

<ResponseField name="error" type="boolean">
  Indicates if an error occurred
</ResponseField>

<ResponseField name="data" type="array">
  Array of sale detail objects for this specific sale
  
  <Expandable title="data properties">
    <ResponseField name="idSales" type="string">
      UUID of the sale order (matches the ID parameter)
    </ResponseField>
    
    <ResponseField name="idSalesDetails" type="string">
      UUID of the sale detail record
    </ResponseField>
    
    <ResponseField name="clientsId" type="string">
      UUID of the client who made the purchase
    </ResponseField>
    
    <ResponseField name="clientsName" type="string">
      Name of the client
    </ResponseField>
    
    <ResponseField name="idProduct" type="string">
      UUID of the product
    </ResponseField>
    
    <ResponseField name="nameProduct" type="string">
      Name of the product
    </ResponseField>
    
    <ResponseField name="quantity" type="integer">
      Quantity purchased
    </ResponseField>
    
    <ResponseField name="price" type="string">
      Unit price of the product at the time of sale
    </ResponseField>
    
    <ResponseField name="total" type="string">
      Total amount for this product (price × quantity)
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

### Invalid ID

```json
{
  "error": true,
  "msg": "id is invalid"
}
```

Returned when the provided ID is not a valid UUID format.

## Understanding Multi-Product Sales

When a sale contains multiple products:
- All items share the same `idSales` value
- Each product has a unique `idSalesDetails` identifier
- The response array contains one object per product
- All items are associated with the same client
